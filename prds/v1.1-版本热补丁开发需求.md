
# 1. 规则修复（short模式）
网站目前有两种模式，分别是classic（长牌 Texas Hold’em）和short（短牌 6+ Hold’em）。本次只需要修复 short 模式下两人对局（Heads-up） 的下注顺序。
*注意：short 模式多人逻辑不动；classic 模式所有逻辑不动。

正确规则如下：
Heads-up 角色关系：
- Button = 小盲位（Small Blind）
- 另一名玩家 = 大盲位（Big Blind）

Preflop：
- 小盲位（Button）先行动
- 大盲位后行动

Flop / Turn / River：
- 大盲位先行动
- Button（小盲位）后行动

请确保：
- 只修改 short 的 heads-up 逻辑
- short 多人模式保持不变
- classic 模式不受影响

# 2. 移动端UI优化
## 2.1 公共牌与Pot布局
现状：当前 Pot 在公共牌上方一行，占用额外空间。
修改为：公共牌和 Pot 在同一行显示。
示例：[Card] [Card] [Card] [Card] [Card]      Pot: 1200
要求：
- 五张公共牌横向排列
- Pot 显示在公共牌右侧
- Pot 不再单独占一行

## 2.2 用户手牌 + 操作按钮 + 下注区域布局
整体要求：
- 这一整块内容（手牌 + 按钮 + 下注输入区域）位于 场上已翻牌公共牌下方、用户池上方
- 保证紧凑展示，尽量占 手机屏幕上半部分
- 删除重复或不重要信息，例如当前是谁在操作的提示只保留一处

### 2.2.1 用户手牌+操作按钮
现状：有单独的操作框，用户的手牌下方有一行操作按钮。
修改为：用户手牌放在左侧，操作按钮放在右侧。操作按钮分为两行，每行两个按钮：
第一行：Check    Call
第二行：Fold     All-in
排列要求：
- 两行按钮高度与手牌区域高度齐平
- 按钮排列紧凑

### 2.2.2 下注区域（Bet Area）
现状：下注区域有单独的操作框，且输入框和按钮分成两行。
修改为：下注输入区域输入框和按钮整合在同一行。
下注区域结构：
[ amount输入框 ]   1/2 Pot   Pot   Confirm

输入框：placeholder 使用浅灰色文字："amount"，提示用户可以在此填写金额，当有金额输入时，文字自动消失；用户可以手动输入金额
快捷下注按钮：
- 1/2 Pot：自动计算 Pot 的一半并填入输入框，但不帮用户下注
- Pot：自动计算当前 Pot 大小并填入输入框，但不帮用户下注
Confirm按钮：用户点击后才真正执行下注

注意：
- 点击 1/2 Pot 或 Pot 只填入金额
- 必须点击 Confirm 才下注
  
按钮颜色要求：
- 1/2 Pot 与 Pot 使用同一类型颜色
- Confirm 使用明显不同的颜色
- 让用户能区分 “计算金额” 和 “执行下注”
  
# 3. 固定布局
用户池以上的内容固定：包含公共牌、Pot、用户手牌、操作按钮、下注输入区域
用户上拉/下拉页面时，这一部分保持不动，仅滑动 用户池以下区域。

# 4. 玩家列表布局
现状：当前玩家列表一行一个玩家，占用空间过多。
修改为：每行显示两个玩家（两列布局）
示例：
PlayerA      PlayerB
PlayerC      PlayerD

# 5. 移动端输入框缩放问题
修改为：点击输入框时浏览器不要自动放大，也不自动下滑页面。

要求：
所有 input，font-size ≥ 16px


# *总体目标
优化后的移动端应满足：
- short 模式两人局翻牌后大盲先行动
- UI 紧凑、信息集中
- 用户池以上内容固定，屏幕上半部分显示完整牌局信息：
  - 公共牌 + Pot
  - 用户手牌 + 操作按钮 + 下注输入区域
- 用户无需滚动即可完成主要操作
- 不影响其他模式和多人局逻辑
